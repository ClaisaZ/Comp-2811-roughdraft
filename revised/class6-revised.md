# Comms / UX / UI Design Redesign Project - Class 6

This week focused on preparing for user testing and learning the fundamentals of color and typography for our high-fidelity wireframes.

## User Testing Preparation 

User testing is an important step in deciding design decisions. It allows designers to observe how real users interact with wireframes and identify usability issues.

## Key Guidelines

- Wireframes should reflect the final design
- Should include:
    - Clear headings and content for context
    - Functional links or navigation indicators
- During testing:
    - Designers cannot guide or help the user
    - Only observe user behavior and take notes
    - Intervene only if something is clearly broken
- In group settings:
    - One person acts as the facilitator(handles navigation)
    - One person acts as the observer(records user behavior)

This process helps find unexpected user actions and scenarios where the design may be confusing.

## High-Fidelity Design Preparation

We will first develop low-fidelity wireframes, working our way into developing our high-fidelity wireframe designs.

High-fidelity designs include:
- Typography(fonts)
- Color schemes
- Visual styling(layout, spacing, images)

## Color Theory

Understanding color is important for creating visually effective and user friendly designs.

**Additive** vs **Subtractive**

- Additive color
    - Starts with black (screen-based)
    - Uses Red, Green, and Blue (RGB)
    - Colors are created by adding light
- Subtractive color
    - Starts with white(paper)
    - Colors are created by removing light
    - Common in print

## Color Requirements

For the high-fidelity design, the following requirements must be met:
- Use a **Three-Color** scheme:
    - Primary color (most used)
    - Secondary color(supporting)
    - Accent color(used for emphasis)
- Color scheme:
    - Cannot be only black and white
    - Must follow a valid **color harmony**
    - Cannot be analogoes or monochromatic
- Accessibility:
    - Must meet at least WCAG AA contrast standards
    - Use tools like [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) to validate color choices.

## Color and Branding

Color should be chosen to intentionally match the target audience and communicate the right message.

Different colors can influence how users perceive a product or brand, so color selection is a very important role in the overall user experience.

## Typography

Typography helps improve readability and helps with visual clarity and hierarchy.

### Key considerations

- Select two fonts:
    - One for headings
    - One for body text
- Fonts should:
    - Be readable and consistent
    - Match the tone of the website
    - Support clear hierarchy
