# Comms / UX / UI Design Redesign Project — Class 8

This week focuses on live user testing using our low-fidelity wireframes. The goal was to observe how users interact with the design and identify usability issues without providing guidance.

These user scenarios are crucial for when we attempt to create our high-fidelity wireframes.

Three users participated in the testing process, each completing two main tasks related to product interaction and event registration.

## Main User Scenario/Flow Tasks

**User Scenario 1:** Subscribing to a product and recieving updates.

User optionally logs in -> User chooses product in navigation -> Selects product of choice -> Clicks subscribe to updates -> Log in if needed -> Confirmation window

**User Scenario 2:** Registering for a in-person event.

User optionally logs in -> User clicks on events -> Selects event of choice -> clicks on more details -> register/book event -> Log in if needed -> Confirmation window

## Observations

**User 1 - Sam**

Task 1 (Product / Subscription):
Sam navigates to the product section but the product that they intended to visit was not there. This shows confusion about the structure of the product page

When attempting to subscribe, Sam was not confident when they were trying to subcribe, telling us that the context and/or labeling was unclear.

Task 2 (Events):
Sam succesffuly navigated through the events section and completed the task without issues.

However, after completeing the process, there was no confirmation or success message, which created uncertainty about whether their action was completed.

**User 2 - Ria**

Task 1 (Product / Subscription):
Ria navigated to the products section and seleceted a product correctly.

She then attempted to subscribe to updates, but there was no indication of whether she was logged in. This created confusion and told us that we need a confirmation window to show feedback to the user.

Also, after subscribing, the system redirected back to the homepage without confirmation, making the action feel incomplete.

Task 2 (Events):
Ria completed the event registration process successfully.

Similar to User 1, the same issue occurred, no confirmation message after completing the action.

**User 3 - Shervin**

Task 1 (Product / Subscription):
Shervin initially clicked the hero image, which redirected directly to a product page. This behavior showed that users may interpret large visual elements as primary navigation.

He then was able to successfully complete the subscription process after logging in.

Task 2 (Events):
Shervin successfully navigated the events section, accessed event information, and completed the registration form without issues.

Sherving also stated that there was no confirmation window after booking an event.

## Key Findings

From the testing sessions, several problems occurred.

- Users were confused about the product structure
- No clear feedback after completing actions
- Users expected confirmation messages or success pages
- Navigation to events were clear and intuitive
- Large visual elements can influence user behavior

## Design Improvements

Based on these findings, the following improvements were identified:
- Add clear leables and context for product subscription features
- Implement confirmation messages or success pages after actions
- Provide authentication feedback(prompt login)
- Ensure consistent user flow after completing actions(avoid redirecting without feedback)
- Review the role of the hero images

## Key Insight

User testing revealed that even small gaps in feedback and clarity can significantly impact the user experience. While navigation was generally successful, missing confirmations and unclear context created confusion.

Addressing these issues will improve usability and make the redesign more intuitive giving us valuable feedback for our high-fidelity wireframes.
