### Week 4 UX/UI Design in-class topics

#### Problem Statements

- Know some ideal users
- Clarify the 5 W's

    - One paragraph description of the problem you are trying to solve by creating/redesigning a site/product/service
    - Answer the 5 M's
        - Who... Who is using the site 
        - What... What is the site offering (what should the main focus be)
        - Where... Online
        - When... ASAP, later?
        - Why... Why create/redesign this thing

#### Feature List

- Scope 
    - What are you going to do/implement for this website redesign

    - Example
        - Update branding (colors, typography)
        - Redesign home page layout, etc
        - redesign menu

#### Ideation

- Generate as many possible solutions to the problem as you can Non-judgemental

- Techniques
    - Brainstorming
    - Worst Possible Idea
    - Crazy8s (genereate quick sketches (wireframes)) 

#### Information Architecture

- Organizing, labeling of context, to help users find information.
- Comprised of several "systems"
    - navigation (what type of menu, how many menus, hierarchy)
        - hierarchy 
            - deep(very few top menu links with a lot of links underneath)
            - shallow(many top menu links with very few links underneath)
        - labelling(what style of language do we use)
        - content inventory(listing all of the pages/content of the website)
            - sitemap(a visual represnentation of all the content)
        - wireframes(simple line drawings depicting the layout of menus/images)
